---

## 1. 프로젝트 목표

**DocFlow AI**는 정부 공고를 자동으로 수집·분석하고, AI가 지원서를 초안 작성하면 사용자가 검토·승인하는 문서 자동화 시스템입니다.

**핵심 플로우:**
```
기업마당 API → 공고 수집 → AI 적합도 분석 → 공고 모니터링 화면 표시
→ 사용자가 "지원서 자동 생성" 클릭 → AI 심층 분석 + 지원서 DOCX 생성
→ 결재함에서 검토 → 승인/반려 → 다운로드
```

---

## 2. 기술 스택

| 레이어 | 기술 |
|--------|------|
| Frontend | Next.js 15 (App Router) · TypeScript · Tailwind CSS |
| 인증 | NextAuth + KakaoProvider + Supabase upsert |
| Backend | Python 3.11 · FastAPI · uv |
| AI | OpenAI GPT-4o / GPT-4o-mini |
| 문서 생성 | python-pptx, openpyxl, python-docx, weasyprint |
| 슬라이드 | Jinja2 HTML + Playwright (선택적) |
| DB | Supabase PostgreSQL |
| 공고 API | 기업마당(bizinfo.go.kr) 오픈 API |
| 배포 | Vercel (Frontend) + Railway (Backend) |

## 3. 완료된 기능

### 인증 & 사용자 관리
- 카카오 OAuth 로그인 (NextAuth KakaoProvider)
- Supabase `users` 테이블 자동 upsert
- JWT → Supabase UUID 연동
- 인증 필요 라우트 미들웨어 보호

### AI 문서 생성
- 자연어 → PPT / Excel / Word / PDF 생성 (OpenAI Function Calling)
- 스트리밍 SSE 응답 (`/api/generate/stream`)
- 파일 업로드 + RAG (청크 → 임베딩 → Supabase 벡터 저장)
- 내 문서 목록 & 삭제

### 슬라이드 에디터
- AI 슬라이드 구조 설계 + HTML 렌더링 (Jinja2)
- 브라우저 내 비주얼 편집 (iframe + postMessage)
- 요소 클릭 → AI 재작성 (PATCH /slides/{id}/rewrite)
- PDF/PPTX 내보내기 (Playwright)

### 워크플로우 관리
- Supabase `workflows` 테이블 CRUD
- 활성화/비활성화 토글

### 정부공고 파이프라인 (구현 완료, 일부 버그)
- 기업마당 API 공고 수집 → AI 요약/점수 → Supabase 저장
- 공고 모니터링 UI (점수 필터, D-day 배지, 키워드)
- 지원서 자동 생성 요청 → 결재함
- 결재함 승인 → AI 심층분석 + 지원서 DOCX 생성
- 승인/반려 UI

### RLS & 보안
- Supabase RLS 정책 (users, documents, workflows)
- `require_auth=True` 운영 기본값
