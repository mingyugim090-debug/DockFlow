# DocFlow AI 개발 인수인계 문서 (For Claude Code & AI Agent)

이 문서는 DocFlow AI 프로젝트의 현재 개발 상태, 구조, 그리고 다음 목표를 AI(Claude 등)가 신속히 파악하고 개발을 이어나갈 수 있도록 작성된 핸드오버(Handover) 문서입니다.

---

## 🏗️ 1. 프로젝트 개요 및 기술 스택
**DocFlow AI**는 사용자의 프롬프트와 참조 문서를 바탕으로 PPT, Word, Excel, PDF(계약서, 제안서) 등의 문서를 자동 생성하고, 대화형 UI를 통해 조율할 수 있는 AI 워크스페이스입니다.

* **Frontend:** Next.js 14 (App Router), React, Tailwind CSS, Lucide React, NextAuth (카카오 로그인 연동)
* **Backend:** FastAPI, Python 3.11+, Uvicorn, Playwright, Jinja2, PyPDF2, python-pptx, openpyxl, python-docx
* **AI & Agent:** OpenAI (Function Calling), Langchain 스타일 자체 `DocumentOrchestrator`
* **DB & Auth:** Supabase (NextAuth callback을 이용해 카카오 provider 사용자 데이터를 Supabase `users` 테이블에 UPSERT 동기화)

## 🎯 2. 현재 개발 완료 현황 (Current Status)

최근까지 가장 핵심적으로 **"Phase 1~5: 웹 브라우저 기반 AI 비주얼 슬라이드 에디터"** 개발을 마쳤습니다. 

### [백엔드 (API & Engine)]
1. **Tool Registry (`backend/agent/tools/`):** 
   - `ppt_tool.py`, `word_tool.py`, `excel_tool.py` 구현 완료
   - `pdf_tool.py` (WeasyPrint/HTML 연동) 완료
   - `search_tool.py` (Tavily 웹 검색 연동) 완료
   - `slide_tool.py` (**New**: 시각적 프레젠테이션 설계 및 요소별 HTML Rewrite 툴) 도입 완료
2. **슬라이드 생성 및 렌더링 엔진 (`backend/slides/`):**
   - **`engine.py`**: Jinja2를 통한 HTML 템플릿(`.html`) 및 테마(`.json`) 조합 컴파일러 구축.
   - **`export.py`**: Playwright로 Chromium 브라우저를 백그라운드 구동시켜 HTML을 캡처하여 PPTX 삽입 및 PDF로 병합 내보내기 구현 (Windows Local file URI 예외 처리 포함).
3. **슬라이드 API 라우터 (`backend/api/routes/slides.py`):**
   - `/generate` (덱 구조 AI 설계 및 생성), `/{deck_id}/slides/{index}/html` (에디터 렌더링용), `/rewrite` (요소 AI 재작성), `/export` (다운로드 변환) 완료.

### [프론트엔드 (UI & Components)]
1. **인증 / 글로벌 레이아웃:** 
   - `사이드바`에 NextAuth 세션을 연동하여 유저 프로필 및 로그아웃 반영 완료.
2. **대시보드 (`app/page.tsx`):**
   - GenSpark 및 Linear 스타일의 프리미엄 다크/화이트 혼합된 레이아웃 리밸런싱.
   - [AI 프레젠테이션] 생성 진입점 추가.
3. **슬라이드 에디터 (`app/slides/[deckId]/page.tsx`):**
   - **`SlideEditor.tsx`**: 백엔드에서 받은 HTML을 `iframe`으로 렌더링하고, 클릭된 요소의 Bounding Box 좌표 및 셀렉터를 탐지하여 보라색 테두리를 띄워줍니다(postMessage 통신).
   - **`SlidePanel.tsx`**: 클릭된 요소의 텍스트가 패널에 표시되며, "글자 크기 키워줘" 등의 버튼을 누를 시 백엔드의 `/rewrite` API를 호출하고 iframe 내의 요소만 Hot-Swap 해줍니다.
   - **`GenerateModal.tsx`**: 홈 대시보드에서 슬라이드 주제와 테마, 슬라이드 장수를 입력받는 모달 화면.

## 🛠️ 4. 구동 방법 (How to run)

**Terminal 1 (Backend - FastAPI)**
```bash
cd backend
.venv\Scripts\activate  # 또는 uv run 활용
uvicorn main:app --reload --port 8000
```
> *참고사항:* playwright 브라우저 모듈이 백엔드 내부 가상 환경에 설치되어 작동합니다.

**Terminal 2 (Frontend - Next.js)**
```bash
cd frontend
npm install
npm run dev  # 3000번 포트 할당
```
> *.env.local*에 Supabase 키 및 카카오 로그인 시크릿이 정상 세팅되어 있는지 확인합니다 (`http://localhost:8000` 로 API Rewrite됨).
